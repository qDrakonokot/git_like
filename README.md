# MyGit — Упрощенная система контроля версий на C

[cite_start]Данный проект представляет собой консольную систему контроля версий (VCS), разработанную на языке C[cite: 2]. [cite_start]Логика работы программы вдохновлена принципами Git и включает в себя продвинутый функционал: управление ветками, умную рекурсивную индексацию путей и работу в режиме отсоединенной головы (detached HEAD)[cite: 23, 27, 35].

## Основные принципы архитектуры

* [cite_start]**Хранение файлов:** Коммиты сохраняют полные копии измененных файлов[cite: 4]. [cite_start]Для неизмененных файлов в коммите сохраняется ссылка на коммит с их последней актуальной версией[cite: 4].
* [cite_start]**Связи в истории:** Каждый коммит в системе обязательно хранит ссылку на свой родительский коммит[cite: 5].
* [cite_start]**Универсальность идентификаторов:** Все команды, принимающие в качестве аргумента хэш коммита (`commit`), также могут принимать имя ветки[cite: 26].
* [cite_start]**Надежность:** Программа корректно обрабатывает невалидные параметры и выводит понятные сообщения об ошибках[cite: 3].

---

## Состояния репозитория

1. [cite_start]**Нормальное состояние:** По умолчанию активна ветка `master` (создается при инициализации)[cite: 23]. Вы находитесь на острие одной из веток.
2. [cite_start]**Состояние «Отсоединенной головы» (Detached HEAD):** Активируется при переходе (`checkout`) на конкретный хэш коммита, а не на имя ветки[cite: 35]. [cite_start]В этом состоянии часть функционала блокируется[cite: 36]. Доступны только команды:
   * [cite_start]`checkout <commit>` [cite: 36]
   * [cite_start]`checkout <branchName>` [cite: 36]
   * [cite_start]`branch <name>` [cite: 36, 37]
   * `switch <branchName>` *(кастомная команда)*
   
   [cite_start]Выход из этого состояния в нормальное происходит при переключении на ветку с помощью команд `checkout`, `switch` или при создании новой ветки[cite: 38].

## Справочник команд

### 1. Инициализация и статус
* [cite_start]`init` — Инициализирует пустой репозиторий в текущей директории, создает ветку `master` и делает начальный пустой коммит[cite: 8, 23]. [cite_start]Если репозиторий уже существует, выводит предупреждение[cite: 9].
* [cite_start]`status` — Показывает список файлов, подготовленных для следующего коммита (созданные, удаленные, измененные)[cite: 19, 20]. [cite_start]Также выводит список файлов, которые были изменены в рабочей директории, но еще не были добавлены в индекс[cite: 31].

### 2. Индексация и коммиты
* [cite_start]`add <path>` — Умное индексирование изменений[cite: 27]. [cite_start]Параметр `<path>` может быть файлом или директорией[cite: 28]. [cite_start]Программа рекурсивно обходит путь и сама определяет, какие файлы были созданы, изменены или удалены (автоматически совмещает функции `add` и `remove`)[cite: 27, 29, 30].
* [cite_start]`commit <message>` — Фиксирует проиндексированные изменения[cite: 12]. [cite_start]Создает коммит, содержащий копии файлов, сообщение, время создания и хэш, вычисленный на основе всей этой информации[cite: 13].

### 3. Просмотр истории и различий
* [cite_start]`log [commit] [--n num]` — Выводит цепочку коммитов от текущего (или указанного) назад до самого первого[cite: 14, 16]. [cite_start]Флаг `--n` позволяет ограничить вывод до `num` коммитов[cite: 15].
* [cite_start]`log <commit1>..<commit2>` — Выводит всю цепочку коммитов строго от `commit2` назад до `commit1`[cite: 32].
* [cite_start]`diff <commit>` — Сравнивает указанный коммит с текущим состоянием[cite: 18]. [cite_start]Выводит список измененных файлов и их хэши[cite: 18].
* [cite_start]`diff <commit1> [commit2]` — Сравнивает два коммита между собой[cite: 39]. [cite_start]Помимо списка файлов, выводит измененные строки внутри них[cite: 39]. [cite_start]Если `commit2` не указан, сравнение идет с текущим коммитом[cite: 40].

### 4. Управление версиями и ветками
* [cite_start]`branch [name]` — Если имя указано, создает новую ветку от текущего коммита[cite: 24]. [cite_start]Если запущено без аргументов, выводит список всех существующих веток в репозитории[cite: 25].
* `switch <branchName>` — Переключает репозиторий на указанную ветку `branchName` и обновляет рабочую директорию.
* [cite_start]`checkout <commit> <file_name>` — Извлекает версию файла `file_name` из указанного коммита/ветки и заменяет им файл в текущей директории[cite: 21].
* [cite_start]`checkout <commit>` — Приводит состояние всей рабочей директории к виду указанного коммита или ветки[cite: 33]. [cite_start]Команда выполнится только в том случае, если в рабочей директории нет незафиксированных изменений[cite: 34].

## Сборка

Для компиляции проекта используется make:

`make`

Для очистки от объектных файлов и исполняемого 

`make clean`
