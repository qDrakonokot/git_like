# MyGit — Упрощенная система контроля версий на C

Данный проект представляет собой консольную систему контроля версий (VCS), разработанную на языке C. Логика работы программы вдохновлена принципами Git и включает в себя продвинутый функционал: управление ветками, умную рекурсивную индексацию путей и работу в режиме отсоединенной головы (detached HEAD).

## Основные принципы архитектуры

**Хранение файлов:** Коммиты сохраняют полные копии измененных файлов.
**Связи в истории:** Каждый коммит в системе обязательно хранит ссылку на свой родительский коммит.
**Универсальность идентификаторов:** Все команды, принимающие в качестве аргумента хэш коммита (`commit`), также могут принимать имя ветки.
**Надежность:** Программа корректно обрабатывает невалидные параметры и выводит понятные сообщения об ошибках.

## Состояния репозитория

**Нормальное состояние:** По умолчанию активна ветка `master` (создается при инициализации). Вы находитесь на острие одной из веток.
**Состояние «Отсоединенной головы» (Detached HEAD):** Активируется при переходе (`checkout`) на конкретный хэш коммита, а не на имя ветки. В этом состоянии часть функционала блокируется. Доступны только команды:
   * `checkout <commit>`
   * `checkout <branchName>`
   * `branch <name>`
   * `switch <branchName>`
   
Выход из этого состояния в нормальное происходит при переключении на ветку с помощью команд `checkout`, `switch` или при создании новой ветки.

## Справочник команд

### 1. Инициализация и статус
* `init` — Инициализирует пустой репозиторий в текущей директории, создает ветку `master` и делает начальный пустой коммит. Если репозиторий уже существует, выводит предупреждение.
* `status` — Показывает список файлов, подготовленных для следующего коммита (созданные, удаленные, измененные).Также выводит список файлов, которые были изменены в рабочей директории, но еще не были добавлены в индекс.

### 2. Индексация и коммиты
* `add <path>` — Умное индексирование изменений. Параметр `<path>` может быть файлом или директорией. Программа рекурсивно обходит путь и сама определяет, какие файлы были созданы, изменены или удалены (автоматически совмещает функции `add` и `remove`).
* `commit <message>` — Фиксирует проиндексированные изменения. Создает коммит, содержащий копии файлов, сообщение, время создания и хэш, вычисленный на основе всей этой информации.

### 3. Просмотр истории и различий
* `log [commit] [--n num]` — Выводит цепочку коммитов от текущего (или указанного) назад до самого первого. Флаг `--n` позволяет ограничить вывод до `num` коммитов.
* `log <commit1>..<commit2>` — Выводит всю цепочку коммитов строго от `commit2` назад до `commit1`.
* `diff <commit1> [commit2]` — Сравнивает два коммита между собой. Помимо списка файлов, выводит измененные строки внутри них. Если `commit2` не указан, сравнение идет с текущим коммитом.

### 4. Управление версиями и ветками
* `branch [name]` — Если имя указано, создает новую ветку от текущего коммита. Если запущено без аргументов, выводит список всех существующих веток в репозитории.
* `switch <branchName>` — Переключает репозиторий на указанную ветку `branchName` и обновляет рабочую директорию.
* `checkout <commit> <file_name>` — Извлекает версию файла `file_name` из указанного коммита/ветки и заменяет им файл в текущей директории.
* `checkout <commit>` — Приводит состояние всей рабочей директории к виду указанного коммита или ветки. Команда выполнится только в том случае, если в рабочей директории нет незафиксированных изменений.

## Сборка

Для компиляции проекта используется make:

`make`

Для очистки от объектных файлов и исполняемого 

`make clean`
